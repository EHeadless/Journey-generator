# Demand Space Rubric

A judge scores generated demand spaces against this rubric. Output is graded
on a 1-5 scale per criterion, weighted, then aggregated. Pass requires
weighted total ≥ 4.0 AND no gate criterion below 3.

## Reference: what a good demand space looks like

Each demand space must:
- Pass the **Remove the Product test** — if you delete the company's product,
  the underlying human motivation still exists.
- Use a 2-4 word evocative **label** (NOT "I want to...").
- Use the **Job to Be Done format**: "When I [situation], I want to [action], so that [outcome]".
- Be **distinct** from siblings — no two demand spaces with the same root motivation.
- Be **phase-specific** — fit the journey phase's description, not generic.
- Avoid product features ("Track my request"), UX requirements ("Seamless flow"),
  or marketing-funnel labels ("Awareness", "Consideration").

## Criteria

| Criterion | Weight | Gate | Description |
|---|---|---|---|
| Remove-the-Product | 0.30 | yes | Every demand space must describe a human life motivation that exists without the company's product. Generic motivations score high; product features or UX requirements score low. |
| Format | 0.15 | yes | Label is 2-4 evocative words (NOT "I want to..."). JTBD strictly follows "When I [situation], I want to [action], so that [outcome]". Mark down 1 point per item that violates. |
| Distinctness | 0.15 | no | Each demand space addresses a distinct motivation. Two spaces with the same underlying job (just reworded) score low. |
| Phase Specificity | 0.15 | no | Motivations match the journey phase context (e.g. Pre-Booking ≠ Post-Visit). Generic motivations that could fit any phase score low. |
| Count | 0.10 | no | 8-12 demand spaces is ideal. Fewer than 6 or more than 14 scores low. |
| Behavioral Language | 0.15 | no | Uses behavioral-strategy vocabulary: tensions, life motivations, situations, struggles. Avoids generic marketing-speak ("personalised journey", "delight customers"). |
